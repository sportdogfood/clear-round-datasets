3. Research rules (EVENT)
--------------------------------------------------

You must follow the global research rules from instructions.txt (Category A/B/C, no fabrication, verification, non-breaking behavior) PLUS the event-specific rules below.

3.1 Where to look

Given:
- event_name
- host_city, host_state
- venue_name
- official_event_link (may be empty)
- official_venue_link

Follow this order:

1) Primary event anchors (preferred):
   - The official event page at {{official_event_link}} if provided.
   - Event-specific pages under the venue’s official domain that clearly describe this event or series.
   - Official prize lists, prize books, or event brochures hosted by the venue or governing body for THIS event or series.

2) Federation / sanctioning anchors:
   - USEF, FEI, USHJA, or other recognized federations listing this competition.
   - Use these for rating labels, sections (hunter/jumper), and any explicit caliber language.

3) Secondary anchors:
   - Reputable equestrian media or series sites that describe the event’s role, caliber, or history.
   - Large, neutral show calendars where this event is listed with rating labels.

3.2 Name and aliases

- Canonical event name:
  - Resolve from the primary event header/logo/title on official or federation pages.
  - If it closely matches event_name from input, use the precise on-page version.

- Aliases:
  - Alternate designations or titles used officially (e.g., with series names, “Spring 1” vs “Spring I”).
  - Include only explicit variants from headers/meta/titles, not from casual references or chatty copy.
  - If an official descriptor like “Finals”, “Championship”, “Classic”, or “Circuit” is part of the branding, you may treat that as part of an alias when it appears.

3.3 Dates

- If an official event page or federation listing gives exact dates for this leg:
  - Use those dates as authoritative start_date and end_date.
  - Convert them to ISO format: YYYY-MM-DD.

- If no event-specific dates are visible:
  - Use span_start_date and span_end_date from the input as fallback.
  - Convert those curated dates (MM/DD/YYYY) to ISO format.

You must always output valid ISO strings for start_date_iso and end_date_iso. If conversion fails for any reason, set both to "could-not-verify".

3.4 Rating and rider caliber

Rating tags:
- Collect ONLY the exact rating/sanction labels that appear in official or federation text for THIS event or series, such as:
  - "USEF Premier 'AA'"
  - "Level 6 Jumper"
  - "Hunter (Regional)"
  - "Jumper (2)"
  - "H/J Channel: 2"
- Store them as an array of strings in rating_tags.
- You may also copy or lightly normalize span_rating / event_rating if they already match the official wording.
- Never infer or upgrade ratings beyond what you see.
- If no explicit ratings are visible beyond the curated hints and those hints cannot be matched to any allowed label:
  - Set rating_tags to ["could-not-verify"] and rating_string_verbatim to "could-not-verify".

Rider caliber:
- Look ONLY for explicit language in official, federation, or high-credibility event descriptions.
- Allowed rider_caliber values:
  - "Olympic"
  - "World-Class"
  - "International"
  - "National"
  - "Regional"
  - "Local"
- Choose the single strongest level that is clearly and explicitly supported.
  - Example: if the event is described as “premier international competition drawing Olympic and World Cup riders,” rider_caliber may be "World-Class".
- If you cannot find clear caliber language:
  - Do NOT infer from rating alone.
  - Set rider_caliber to "could-not-verify".

3.5 Known-for and prestige tone

Known_for:
- One concise clause describing what the event is known for, grounded in sources.
- Examples:
  - "multi-week spring series that anchors the regional hunter-jumper calendar"
  - "indoor championship leg that closes the fall season"
- Must be supported by at least one allowed source.
- If there is no clear, repeated theme in reliable text, set known_for_clause to "could-not-verify".

Prestige tone:
- Short clause capturing the prestige tone from official positioning:
  - e.g., “marketed as a high-profile stop for serious hunter-jumper programs”.
- Must be grounded in phrases like “premier”, “flagship”, “major”, “world-class” found in official/federation copy.
- If such language is absent or ambiguous, set prestige_tone_clause to "could-not-verify".

Field feel:
- Short clause describing the type of rider field this event tends to draw, using supported cues:
  - “mix of professionals, amateurs, and juniors in a multi-ring format”
  - “national-level riders and qualifying amateurs”
- Do NOT infer from rating alone.
- If not supported, set field_feel_clause to "could-not-verify".
